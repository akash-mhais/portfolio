01 // Polling & Trigger

The system operates on an asynchronous 100ms polling loop. It continuously monitors the PLC data table for a **Rising Edge** on Bit 0. This ensures that the inspection cycle is only triggered once per mechanical event, preventing double-processing.

02 // Payload Mapping

Simultaneously with the trigger, the bridge reads the **Program Selection DINT**. This 23-bit value is decoded to load specific component templates, ROI coordinates, and sensitivity thresholds from the Vision Studio library.

03 // Handshake Cycle

Once the Vision Engine returns a result, the Communicator updates the PLC's Output table. It pulses the **"Inspection Complete"** bit for exactly 1000ms. If the handshake isn't acknowledged within this window, the system enters a retry-loop or fails-safe.

04 // System Heartbeat

A dedicated **Heartbeat Bit** toggles every 500ms between the PC and PLC. If the software bridge stops responding, the PLC detects the static bit and immediately halts the line, ensuring that no un-inspected components pass through.
